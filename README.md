# Eventa Access Software-Webshop

Statischer, barrierearm aufgebauter Software-Webshop für die Übung in Daten- und Informatikrecht.

## Projektplan

1. Produkt definieren: Eventa Access ist eine fiktive Software-as-a-Service-Plattform für barrierearme Eventbuchungen, Eventverwaltung und Zugangsinformationen.
2. Unternehmen definieren: Eventa Access GmbH, Wien, 9 Mitarbeitende.
3. Geschäftsmodell definieren: Monatliche SaaS-Pakete für Veranstalter plus optionale Ticketprovision pro Buchung.
4. Website umsetzen: Startseite/Shop, Events, About/Marke, Barrierefreiheit, Urheberrecht, Lizenz, Datenschutz, Kontakt, Impressum.
5. Rechtliche Nachweise sichtbar machen: Medienlizenzen, Nizza-Klassen, Markenrecherche, Nutzungsrechte, DSGVO-Prozesse, Mediengesetz-Impressum.
6. Barrierefreiheit prüfen: Tastatur, Kontrast, Alternativtexte, Labels, Fokus, Statusmeldungen und reduzierte Bewegung.
7. Deployment: Repository auf GitHub hochladen und GitHub Pages für den Branch `main` aktivieren.

## Wichtigster Schwerpunkt für Daten- und Informatikrecht

Die Website soll nicht nur wie ein Demo-Shop aussehen. Entscheidend ist, dass jede rechtlich relevante Entscheidung dokumentiert ist: Welche Daten werden verarbeitet, welche Medien dürfen verwendet werden, welche Rechte bekommt der Kunde, welche Rechte bleiben beim Unternehmen, welche Marken werden benutzt und welche gesetzlichen Informationspflichten werden erfüllt.

## Lokale Nutzung

`index.html` direkt im Browser öffnen. Ein Backend ist nicht erforderlich; Warenkorb, Checkout, Cookies, Sound-Demo und Kontaktformular sind gemockt.
